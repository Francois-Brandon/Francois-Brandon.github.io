CSS3 Transitions and Animations and triggering them with JavaScript

Transition Properties

Below is a table which defines the various properties available to a transition. CSS transitions allow for a smooth change from one property value to another. The only transition property required for the transition to occur is duration. By default the duration is 0s, so there would be no transition unless specified. When defining the transition property, multiple properties can be included by separting them with a comma.

Property Definition Syntax
transition Allows for shorthand definition of the following properties. Duration should always be specified transition: property duration timing-function delay|initial|inherit;
transition-delay Specifies a delay (in seconds) for the transition effect transition-delay: time|initial|inherit;
transition-duration Specifies the length of time (in seconds) the transition effect will take transition-duration: time|initial|inherit;
transition-property Specifies the name of the CSS property that the transition effect is for transition-property: none|all|property|initial|inherit;
transition-timing-function Specifies the speed curve for the transition effect transition-timing-function: linear|ease|ease-in|ease-out|ease-in-out|
step-start|step-end|steps(int,start|end)|cubic-bezier(n,n,n,n)|initial|inherit;

Transition-timing-function has several values available to be used.

Value Definition Syntax
ease This is the default value. The transition will start slow, go fast, then finish slow transition-timing-function: ease;
linear The transition speed will be consistent from start to end transition-timing-function: linear;
ease-in The transition will start slow then maintain a consistent speed transition-timing-function: ease-in;
ease-out The transition will begin at a consistent speed then slow down at the end transition-timing-function: ease-out;
ease-in-out The transition will have both a slow start and a slow end transition-timing-function: ease-in-out;
steps This defines a stepping function that takes two parameters. The first parameter should be a positive integer.
This specifies the number of intervals. The second parameter is optional and can be either start or end.
This parameter indicates when in the interval the change of value should occur.		 transition-timing-function: steps(int,start|end);
step-start Equivalent to steps(1, start) transition-timing-function: step-start;
step-end Equivalent to steps(1, end) transition-timing-function: step-end;
cubic-bezier This defines a Cubic Bezier curve. The curve is defined by four points, p0 to p3. We are only concerned
with the points p1 and p2 as p0 will always be 0,0 and p3 will always be 1,1. The cubic-bezier function
takes four parameters which are the coordinates for two points. The first two parameters are the x and y
coordinates of p1 and the second are the x and y coordinates for p2. A curve will be created based on
those points. The curve will then define the timing of the transition.		 transition-timing-function: cubic-bezier(x1,y1,x2,y2);

Animation Properties

Property Definition Syntax
animation Allows for shorthand definition of the following properties. Duration should always be specified. animation: name duration timing-function delay iteration-count direction fill-mode play-state;
animation-delay Specifies a delay (in seconds) for the animation animation-delay: time|initial|inherit;
animation-direction This specifies if the animation should be played forwards, backwards, or in alternate cycles. There are several property values available.
  • normal is the default value, and will play the animation forwards.
  • reverse will play it backwards.
  • alternate will play forwards first, then backwards.
  • alternate-reverse will play backwards firts, then forwards.
 animation-direction: normal|reverse|alternate|alternate-reverse|initial|inherit;
animation-duration This specifies how long the animation should take to complete. animation-duration: time|initial|inherit;
animation-fill-mode This specifies a style for the element before the animation starts and/or after it plays. Possible values:
  • none is the default value, animation will not apply any styles before or after it runs
  • forwards will cause the element to keep the style values set by the last keyframe after the animation is complete
  • backwards will case the element to keep the style values that are set by the first keyframe. The style will be retained during any delay period.
  • both will implement the behavior of both forwards and backwards
 animation-fill-mode: none|forwards|backwards|both|initial|inherit;
animation-iteration-count This specifies the number of times the animation should be played. The default value is 1, meaning the animation will be played through a single time. The value infinite can also be used in place of a number to indicate that the amimation should play indefinitely. animation-iteration-count: number|infinite|initial|inherit;
animation-name This is used to specify a name for the @keyframes animation. The default value is none, meaning there will be no animation. animation-name: keyframename|none|initial|inherit;
animation-play-state This property specifies the state of the animation, whether it is running or paused. The default value is running. animation-play-state: paused|running|initial|inherit;
animation-timing-function Specifies the speed curve for the animation. The possible values are the same as transition-timing-function. The default value is ease. animation-timing-function: linear|ease|ease-in|ease-out|ease-in-out|step-start|step-end|steps(int,start|end)|cubic-bezier(n,n,n,n)|initial|inherit;
