HTML5 Tags - Video, Audio, Canvas

Video Tag

The <video> tag is used to specify a video, movie clip, or other video stream. There are currently 3 supported MIME types for video formats. Support differs between browsers.

Format MIME-type Specs
MP4 video/mp4 MPEG 4 files with H264 video codec and AAC audio codec
WebM video/webm WebM files with VP8 video codec and Vorbis audio codec
Ogg video/ogg Ogg files with Theora video codec and Vorbis audio codec

The following table contains the various optional attributes available to the <video> tag.

Attribute Value Information
autoplay autoplay When this attribute is present, the video will automatically start playing as soon as it can.
controls controls When this attribute is present, video controls will be displayed. These video controls are:
  • Play
  • Pause
  • Seeking
  • Volume
  • Fullscreen Toggle
  • Captions/Subtitles (when available)
  • Track (when available)
height pixels Defines, in pixels, the height of the video player.
loop loop When this attribute is present the video will replay each time it finishes playing.
muted muted The audio will be muted when the video is played.
poster URL Th image specified here will be shown while the video is loading and until the play button is pressed.
preload auto
metadata
none		 Used to specify how the video should be loaded when the page loads. This attribute is ignored if autoplay is present.
  • auto: The browser should load the entire video when the page loads
  • metadata: Only the metadata should be load when the page loads
  • none: The browser should not load the video when the page loads
src URL Specifies the source URL of the video file.
width pixels Defines, in pixels, the width of the video player.

Video Examples

To specify a video source you can use either the src attribute or the source element. The following example uses multiple source elements to specify multiple formats. This will allow for greater compatibility across browsers.

Autoplay with Loop

<video muted autoplay loop width="600" poster="Ski-Day.jpg">
<source src="Ski-Day.webm" type="video/webm"/>
<source src="Ski-Day.mp4" type="video/mp4"/>
</video>

No Preload with Controls

<video muted controls preload="none" width="600" poster="Ski-Day.jpg">
<source src="Ski-Day.webm" type="video/webm"/>
<source src="Ski-Day.mp4" type="video/mp4"/>
</video>

Audio Examples

Canvas Tag

The <canvas> element is used to draw graphics on a webpage. These graphics can be drawn on the fly using JavaScript. There are several built-in methods that allow for drawing paths, text, and shapes like circles and rectangles. These graphics can even be animated. The canvas element only has two attributes, height and width. The default size is 300 pixels wide and 150 pixels high, but these dimensions can be specified using the attributes. The canvas element can be styled like an image by defining margin, border, background. If no style is defined the canvas element will be transparent. The canvas is a two dimensional grid that uses a coordinate system. The origin (0,0) is located in the top left corner. These coordinates are used to do the actual drawing as will be seen in the examples. To start using a canvas element you must first create one.

<canvas id="mycanvas" height="300" width="500"></canvas>

To draw on this blank canvas a script must be used. The script first needs to access the rendering context. This is done by calling the method getContext(). This method takes one parameter, the context. For 2-dimensional graphics, that would simply be "2d". The getContext() returns an object that has many properties and methods available to draw on the canvas. Our script would start like this

var canvas = document.getElementById('mycanvas');
var ctx = canvas.getContext('2d');

The following table describes some of the available methods and properties available to the getContext("2d") object.

Color, Style, Shadow

Property Value Description

Default value: #000000

Default value: #000000

Default value: 0

Method Syntax Description
createLinearGradient() context.createLinearGradient(x0,y0,x1,y1); Creates a linear gradient object. This can be used to fill rectangles, circles, lines, text, etc.
  • x0: The x-coordinate of the start point of the gradient
  • y0: The y-coordinate of the start point of the gradient
  • x1: The x-coordinate of the end point of the gradient
  • y1: The y-coordinate of the end point of the gradient
createPattern() context.createPattern(image,"repeat|repeat-x|repeat-y|no-repeat"); Repeats the specified element in the specified direction. The element can be an image, video, or another <canvas> element.
  • image: Specifies the image, canvas, or video element of the pattern to use
  • repeat: Default. The pattern repeats both horizontally and vertically
  • repeat-x: The pattern repeats only horizontally
  • repeat-y: The pattern repeats only vertically
  • no-repeat: The pattern will be displayed only once (no repeat)
createRadialGradient() context.createRadialGradient(x0,y0,r0,x1,y1,r1); Creates a radial/circular gradient object. The gradient can be used to fill rectangles, circles, lines, text, etc.
  • x0: The x-coordinate of the starting circle of the gradient
  • y0: The y-coordinate of the starting circle of the gradient
  • r0: The radius of the starting circle
  • x1: The x-coordinate of the ending circle of the gradient
  • y1: The y-coordinate of the ending cirlce of the gradient
  • r0: The radius of the ending circle
addColorStop() gradient.addColorStop(stop,color); Specifies the colors and position in a gradient object, and is used together with createLinearGradient() or createRadialGradient().
  • stop: A value between 0.0 and 1.0 that represents the position between start and end in a gradient
  • A CSS color value to display at the stop position

Rectangles

Method Syntax Description
rect() context.rect(x,y,width,height); Creates a rectangle.
  • x: The x-coordinate of the upper-left corner of the rectangle
  • y: The y-coordinate of the upper-left corner of the rectangle
  • width: The width of the rectangle, in pixels
  • height: The height of the rectangle, in pixels
fillRect() context.fillRect(x,y,width,height); Draws a "filled" rectangle
  • x: The x-coordinate of the upper-left corner of the rectangle
  • y: The y-coordinate of the upper-left corner of the rectangle
  • width: The width of the rectangle, in pixels
  • height: The height of the rectangle, in pixels
strokeRect() context.strokeRect(x,y,width,height); Draws a rectangle (no fill)
  • x: The x-coordinate of the upper-left corner of the rectangle
  • y: The y-coordinate of the upper-left corner of the rectangle
  • width: The width of the rectangle, in pixels
  • height: The height of the rectangle, in pixels
clearRect() context.clearRect(x,y,width,height); Clears the specified pixels within a given rectangle
  • x: The x-coordinate of the upper-left corner of the rectangle
  • y: The y-coordinate of the upper-left corner of the rectangle
  • width: The width of the rectangle, in pixels
  • height: The height of the rectangle, in pixels

Paths

Method Syntax Description
fill() context.fill(); Fills the current drawing (path). The default color is black.
stroke() context.stroke(); Actually draws the path you have defined with moveTo() and lineTo() methods. The default color is black.
beginPath() context.beginPath(); Begins a path, or resets the current path
moveTo() context.moveTo(x,y); Moves the path to the specified point in the canvas, without creating a line
  • x: The x-coordinate of where to move the path to
  • y: The y-coordinate of where to move the path to
closePath() context.closePath(); Creates a path from the current point back to the starting point
lineTo() context.lineTo(x,y); Adds a new point and creates a line to that point from the last specified point in the canvas
  • x: The x-coordinate of where to create the line to
  • y: The y-coordinate of where to create the line to
clip() context.clip(); Clips a region of any shape and size from the original canvas
quadraticCurveTo() context.quadraticCurveTo(cpx,cpy,x,y); Creates a quadratic Bézier curve
  • cpx: The x-coordinate of the Bézier control point
  • cpy: The y-coordinate of the Bézier control point
  • x: The x-coordinate of the ending point
  • y: The y-coordinate of the ending point
bezierCurveTo() context.bezierCurveTo(cp1x,cp1y,cp2x,cp2y,x,y); Creates a cubic Bézier curve
  • cp1x: The x-coordinate of the first Bézier control point
  • cp1y: The y-coordinate of the first Bézier control point
  • cp2x: The x-coordinate of the second Bézier control point
  • cp2y: The y-coordinate of the second Bézier control point
  • x: The x-coordinate of the ending point
  • y: The y-coordinate of the ending point
arc() context.arc(x,y,r,sAngle,eAngle,counterclockwise); Creates an arc/curve (used to create circles, or parts of circles)
  • x: The x-coordinate of the center of the circle
  • y: The y-coordinate of the center of the circle
  • r: The radius of the circle
  • sAngle: The starting angle, in radians (0 is at the 3 o'clock position of the arc's circle)
  • eAngle: The ending angle, in radians
  • counterclockwise: Optional. Specifies whether the drawing should be counterclockwise or clockwise. False is default, and indicates clockwise, while true indicates counter-clockwise.
arcTo() context.arcTo(x1,y1,x2,y2,r); Creates an arc/curve between two tangents
  • x1: The x-coordinate of the first tangent
  • y1: The y-coordinate of the first tangent
  • x2: The x-coordinate of the second tangent
  • y2: The y-coordinate of the second tangent
  • r: The radius of the arc
isPointInPath() context.isPointInPath(x,y); Returns true if the specified point is in the current path, otherwise false
  • x: The x-coordinate to test
  • y: The y-coordinate to test

Canvas Transformations

Transformations can also be applied to <canvas> elements. The methods used to do so are described below.

Method Syntax Values Description
scale() context.scale(scalewidth,scaleheight);
  • scalewidth: Scales the width of the current drawing (1=100%, 0.5=50%, 2=200%, etc.)
  • scaleheight: Scales the height of the current drawing (1=100%, 0.5=50%, 2=200%, etc.)
 Scales the current drawing bigger or smaller
rotate() context.rotate(angle); angle: The roatation angle in radians. Rotates the current drawing
translate() context.translate(x,y);
  • x: The value to add to horizontal (x) coordinates
  • y: The value to add to vertical (y) coordinates
 Remaps the (0,0) position on the canvas
transform() context.transform(a,b,c,d,e,f);
  • a: Horizontal scaling
  • b: Horizontal skewing
  • c: Vertical skewing
  • d: Vertical scaling
  • e: Horizontal translate
  • f: Vertical translate
 Replaces the current transformation matrix for the drawing
setTransform() context.setTransform(a,b,c,d,e,f);
  • a: Horizontal scaling
  • b: Horizontal skewing
  • c: Vertical skewing
  • d: Vertical scaling
  • e: Horizontal translate
  • f: Vertical translate
 Resets the current transform to the identity matrix. Then runs transform()
