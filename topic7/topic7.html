Manipulating CSS Class Properties Using JavaScript

Update/Add CSS Properties

Using HTML DOM you can select specific elements and update its style properties. In the following example the buttons can be used to update certain properties of the div they are located in. By clicking the Update Style button, a function will be called which will the width, border, and background of the div. Clicking on Revert Style will return those style properties to the initial values. Instead of updating the actual class that is associated with the div this will add the style attribut directly to the HTML element. This inline style will take priority over the class properties in the external style sheet. The revert is execute by removing the style attribute from the div element.

One thing to note is how some style properties are referenced using DOM. For example, when defining the background color in a class you do so using background-color. To reference that property we cannot use element.style.background-color. This will try to subtract color from element.style.background. Instead these properties should be written in CamelCase. The previous example would be written element.style.backgroundColor

Click the buttons to update the style of this box. If you are unhappy with the results you may return to the original style.

Update Class

The following examples use getElementsByClassName() which returns a NodeList that is "live". This means that as an element is updated it may be removed from the NodeList. This can cause issues when tring to loop through the list. To account for the "live" nature of this NodeList you can loop backwards starting with the last index.

The Update Class button will retrieve a NodeList for all the elements that have the "codeex" class. It will then loop through and update the className propert with the "newcodeex" class. The Revert Class button will update the class for those elements back to "codeex"

Another method to accomplish a similar task is querySelector() and querySelectorAll(). With these two methods you can return all or a single element in the document that matches a specified class. Where these methods differ from getElementsByClassName() is that you can specify any CSS selector or multiple selectors as a string. Multiple selectors are comma separated. When using the querySelectorAll() the matches are returned as a NodeList. Just as in the previous example, you can loop through the list and update all elements or a single element by specifiying the index.
